# Mermaid 语法修复助手

## 目标

提升 Mermaid 在线编辑器的编辑效率：当实时渲染遇到语法错误时，帮助用户定位当前第一个错误、理解其原因，并取得可安全复用的正确语法参照，从而更快恢复有效预览。

## 背景与约束

- 项目是浏览器本地运行的 Vite + React Mermaid 编辑器；不引入后端、云端存储或协作能力。
- 当前已具备 CodeMirror 编辑、22 类模板、实时渲染、浏览器 localStorage 持久化、Markdown 导入和 SVG/PNG/JPG/Markdown 导出。
- 使用 `mermaid@11.4.1`；规则的正确性以该锁定依赖的本地解析行为为准。Mermaid 官方文档仅用于理解语法类别，不能替代本地回归测试。
- 现有渲染在用户停止输入约 280ms 后触发。新能力必须沿用该节奏，且不得破坏异步渲染的“仅最新请求可提交结果”约定。
- 所有处理和源码持久化继续留在浏览器本地。

## 范围

### 包含

1. 自动诊断当前第一个 Mermaid 语法错误；修复后自动重新诊断后续错误。
2. 将可识别错误定位到编辑器行，并提供简短中文解释、手动修复提示及可展开、可复制的 Mermaid 原始错误详情。
3. 在编辑器底部提供可展开的错误助手；预览区仅保留错误状态，不重复提供修复操作。
4. 为当前模板库的 22 类图表各实现 2 条专属、可测试的错误规则，共 44 条；其他 Mermaid 语法保留通用诊断。
5. 每条专属规则提供 ASCII 标识符加中文可读标签的最小合法片段，并提供“插入到错误行后”和“插入到光标处”两个明确操作。
6. 覆盖桌面及 390px 移动视口的错误、定位和插入体验。

### 不包含

- 自动、静默或经确认后替换、删除、重排用户已有源码。
- 一次性猜测或列出多个语法错误。
- Mermaid 官方全部方言、未来新增方言或模板库之外图表的专属片段。
- AI 生成、服务器端分析、账户、协作和云端保存。

## 交互要求

- 当代码有效时，错误助手收起，渲染和导出行为保持不变。
- 当代码无效时，只处理解析器当前返回的第一个错误。用户修正后重新执行诊断，不显示不可靠的后续错误猜测。
- 错误助手默认显示：错误行（若可确定）、中文解释和“请手动修改高亮错误行”的提示。
- Mermaid 原始错误位于可展开的技术详情中，用户可一键复制。
- “定位错误”必须聚焦并滚动到对应行，且不修改源码。
- 两个插入按钮仅追加片段：一个在错误行后，一个在当前光标处；不得替换选择区或已有文本。对于必须修改原有错误行的情形，片段只作为参照，不承诺点击后立即恢复预览。
- 错误助手和按钮应有可访问名称；键盘用户能够执行定位、展开技术详情、复制和两种插入动作。

## 验收标准

- [ ] 合法 Mermaid 输入仍在停止输入约 280ms 后正常预览；错误被修正后，错误助手自动收起且导出恢复可用。
- [ ] 无效输入只展示当前第一个错误；有可用位置时可定位到正确的 1-based 行，无法可靠确定位置时明确显示通用诊断而不伪造行号。
- [ ] 22 类模板图表各有 2 条专属规则；自动化测试验证规则目录恰为 44 条、每类恰为 2 条，并验证每条规则的匹配、中文解释、ASCII 加中文标签的片段和预期插入位置。
- [ ] 模板范围外或不能匹配专属规则的错误仍显示通用中文说明及可复制的原始错误，不提供错误的专属片段。
- [ ] 定位不会更改源码；两种插入操作只追加片段，且不会覆盖选择区或已有内容。
- [ ] 旧的异步渲染防抖、陈旧请求保护、本地保存、模板加载、设置、Markdown 导入和所有导出路径保持不变。
- [ ] 新增状态和交互有 Vitest 覆盖；`npm test -- --run`、`npm run lint`、`npm run build` 均通过。
- [ ] Chrome 实测合法输入、错误定位、技术详情复制、两种插入、逐个错误重新诊断，以及 390px 视口无横向溢出。

## 开放问题

无。规则的具体匹配样本和片段由技术设计中定义的 44 条目录及本地解析测试确定。
