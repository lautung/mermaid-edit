# Mermaid 在线编辑器 PRD

## 1. 背景

Mermaid 是面向开发者、产品经理、架构师和技术写作者的文本化图表语法。参考 [MermaidEditor](https://mermaideditor.com/zh-hans) 的产品形态，本项目应提供一个轻量、免登录、代码优先的在线 Mermaid 编辑器，让用户可以快速编写、预览、复制和导出图表。

当前项目已经具备基础编辑、实时预览、主题切换、缩放、复制 SVG，以及导出 SVG/PNG/JPG 的能力。本 PRD 用于明确下一阶段产品化方向，而不是替代技术设计或实现计划。

## 2. 产品目标

1. 降低 Mermaid 图表创建门槛，让用户在打开页面后 10 秒内开始编辑并看到预览。
2. 提供可靠的本地渲染和导出能力，满足文档、PPT、博客、需求说明和技术设计中的图表使用。
3. 形成可扩展的工具型产品基础，后续支持模板、格式转换、图表库和协作能力。

## 3. 目标用户

- 开发者：绘制流程图、时序图、类图、状态图、Git 图、ER 图等技术图表。
- 产品经理：绘制用户流程、业务流程、需求关系图和路线图。
- 技术写作者：为 Markdown、博客、知识库和说明文档生成可导出的图表。
- 教学或培训人员：快速生成可复制、可展示、可下载的示例图。

## 4. 核心使用场景

### 4.1 快速编辑并预览

用户打开网页后，左侧输入 Mermaid 代码，右侧实时渲染图表。语法正确时展示图表，语法错误时展示明确错误状态，避免用户不知道失败原因。

### 4.2 导出图表

用户完成图表后，可以导出为 SVG、PNG、JPG，用于插入文档、PPT、邮件、博客或 issue。导出按钮只应在图表可渲染时可用。

### 4.3 复制 SVG

用户可以一键复制 SVG，用于粘贴到支持 SVG 的设计工具、文档系统或代码文件中。

### 4.4 使用模板起步

用户可以从常见图表模板开始，而不是从空白编辑器开始。模板需要覆盖流程图、时序图、类图、状态图、甘特图、饼图、ER 图、用户旅程图等高频类型。

### 4.5 本地隐私处理

用户输入的图表内容默认只在浏览器本地处理，不上传服务器。产品界面需要明确体现该原则，增强用户对私有业务流程和架构图的信任。

## 6. 非功能需求

| 类别 | 要求 |
| --- | --- |
| 性能 | 常规图表编辑后 500ms 内反馈渲染状态；复杂图表允许延迟但必须展示处理中状态。 |
| 可靠性 | 无效 Mermaid 代码不得触发下载空文件或损坏文件。 |
| 隐私 | 默认不上传用户输入内容；如后续加入云端功能，必须显式告知。 |
| 兼容性 | 支持最新版 Chrome、Edge、Firefox、Safari。 |
| 响应式 | 桌面优先，移动端至少可编辑、预览和导出，不出现控件重叠。 |
| 可访问性 | 关键按钮有可读标签；禁用状态有明确视觉反馈。 |
| 部署 | 保持静态前端可部署，适配 Vercel 静态构建。 |

## 7. MVP 范围

MVP 应包含：

1. Mermaid 源码编辑。
2. 实时预览。
3. 错误状态提示。
4. 主题切换。
5. 预览缩放。
6. 导出 SVG、PNG、JPG。
7. 复制 SVG。
8. 本地保存上次编辑内容。
9. 默认示例和重置能力。

MVP 不包含：

1. 登录、云端保存、团队协作。
2. AI 生成 Mermaid。
3. 批量转换。
4. 后端 API。
5. 付费订阅。

## 9. 用户体验要求

1. 首屏必须直接是可用编辑器，不做营销落地页。
2. 编辑区和预览区在桌面端并排展示，移动端可上下排列或用标签切换。
3. 导出、复制、主题、缩放等高频工具集中在工具栏，避免分散。
4. 语法错误时，预览区要说明当前无法渲染，并保留编辑器可操作。
5. 所有导出按钮在不可导出时禁用，不让用户进入失败路径。
6. 不使用需要用户登录才能完成核心流程的设计。

## 10. 验收标准

1. 用户打开页面后可以立即看到示例源码和对应图表预览。
2. 修改 Mermaid 源码后，预览会自动更新。
3. 输入非法 Mermaid 代码后，页面展示错误状态，导出按钮不可用。
4. 恢复合法 Mermaid 代码后，预览恢复，导出按钮重新可用。
5. SVG、PNG、JPG 导出的文件都可以正常打开，内容与预览一致。
6. 复制 SVG 后，剪贴板内容是可用 SVG。
7. 刷新页面后，用户上次编辑内容仍然存在。
8. 在桌面和移动视口下，工具栏、编辑区、预览区和状态信息不重叠。

## 11. 指标

| 指标 | 目标 |
| --- | --- |
| 首次可交互时间 | 3 秒内 |
| 示例图首屏渲染成功率 | 99% 以上 |
| 导出成功率 | 99% 以上，不含浏览器权限限制 |
| 用户完成首次导出路径 | 3 步内：编辑、预览、导出 |
| 非法输入恢复路径 | 修改代码后自动恢复，无需刷新 |

## 12. 风险与约束

1. Mermaid 语法和渲染能力随版本变化，需要在升级依赖时回归常见图表类型。
2. SVG 转 PNG/JPG 容易遇到浏览器 canvas 污染、字体缺失和尺寸截断问题，导出逻辑需要真实浏览器验证。
3. 复杂图表可能导致渲染慢或布局过大，需要考虑防抖、加载状态和预览容器滚动。
4. 若后续加入云端或 AI 功能，隐私承诺需要重新审视，不能沿用“完全本地处理”的表述。
